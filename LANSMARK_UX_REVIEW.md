# LANSMARK UX·흐름 종합 검토 (2026-06-03)

> 방법: 12 전문 페르소나 × 8 시나리오 = **96 검토 패스**(247 유효 발견). 적대적 검증 단계는 세션 토큰 한도로 중단되어, **구현자가 코드 대조로 직접 판정**(파일·라인 확인). 모순 1건(“음수 소득은 데모에서 안 뜸”)은 런타임 측정(블루베리 3,300㎡ → P50 −24,174,400원)과 배치되어 폐기.
> 발견 분포: 출하 80 · 작물산정 64 · 전반UX 60 · 생육 43 · 농지 18 (high 143 / medium 113 / low 11).

---

## 1. UX 흐름 자연스러움 — 판정: **부분적 (핵심 지점에서 단절)**
- **양호**: 필지 클릭 → 토지유형 분기(농경지 재확인 / 수면 차단 / 도시 경고) → 무료 추천 → 소득 시뮬(P10/50/90·근거·면책)까지는 매끄럽고 가드레일도 잘 강제됨.
- **단절**: 흐름이 **소득 카드에서 끝난다**. "그래서 언제 심고·언제 수확해 언제 파나"(생육·출하)가 도달 가능한 화면에 한 줄도 없다. 농부의 실제 의사결정 순서가 끊김.
- **진입 마찰**: 시작 줌 z=7(시·도 단계) → 소득 시뮬은 z>12(필지)에서만 → **수동 2단계 줌인을 모르면 안내문대로 해도 ③소득시뮬이 안 나옴**. 주소/지번 검색·‘내 위치’ 없음 → 초보는 회색 OSM에서 자기 밭을 못 찾음(/api/geocode는 구현됐으나 미배선).

## 2. 농지 → 작물산정 → 생육 → 출하 — 판정: **앞 2단계만 사용자 도달, 뒤 2단계 단절(dead path)**

| 단계 | 코어 모듈 | /api | 프론트(사용자 도달) |
|---|---|---|---|
| **농지**(토지유형) | ✅ `geo/landClass.ts` | ✅ `/api/landclass` | △ (필지 z>12에서만 — z=7에서 강/바다/도시 탭하면 분기 없이 시드작물) |
| **작물산정**(추천·소득) | ✅ `cropSuitability`·`parcelSimulator` | ✅ `/api/recommend`·`/api/simulate` | ✅ |
| **생육**(파종·재배캘린더·생육리스크·병해충) | ✅ `core/{calendar,planting,growthRisk}.ts`·`data/cropCalendar.seed`·`cropPests.seed` | ❌ | ❌ **레거시 `core/simulator.ts`(사용 금지)·자기 테스트에서만 호출 = 죽은 코드** |
| **출하**(수확시기·시장가 타이밍) | ⚠️ 수확월 데이터(`cropCalendar.seed`)만, 타이밍 로직 부재 | ❌ | ❌ (KAMIS 단가는 ‘오늘−30일’ 고정, 수확월과 무연결) |

→ **4단계 중 사용자가 닿는 건 2단계까지.** 생육은 **로직·시드가 이미 완성돼 있는데 canonical 경로(parcelSimulator→/api/simulate→app)에 연결만 안 됨** — 가장 아까운 부분. 출하는 데이터 일부만 있고 로직·UI 부재. 제품 정체성(“작물·**수확**·소득 시뮬레이터”)의 수확 축이 비어 있음.

## 3. 계획 대비 구현 정도
- **달성(~핵심)**: map-first atlas(C1~C6)·P10/50/90 소득·6축 근거·손익분기·신뢰도·면책·출처·실측 보정 플라이휠·토지유형 구분 — 마스터 플랜의 무료추천→유료 정밀시뮬 골격은 구현·검증됨(테스트 102, 가드레일 통과).
- **미달**:
  - **생육·출하 미노출**(위 2항) — 제품 서사의 절반.
  - **입력은 받지만 엔진이 버리는 값들**(“정밀”의 실속 훼손):
    - `salesChannel`(판로) — UI 5지선다·전송·검증 통과하나 `rdaIncome.ts:30`이 단가를 `.mixed`로 하드코딩 → 판로가 소득에 무영향.
    - `targetYear`(재배 연차) — `getRdaBase`가 `yieldKgPerM2ByYear.mature`만 사용 → 다년생(사과·블루베리·포도·도라지) **정착 2~4년 음(−)현금흐름이 첫 소득에서 통째로 은폐**, 손익분기 낙관 편향.
    - **실필지 면적** — 폴리곤은 그리는데 추천·첫 시뮬은 `area:3300`(1,000평) 하드코딩 → 내 밭이 600평이든 2,000평이든 첫 숫자가 동일(소득은 면적 정비례).
  - **무료 추천 위치 무반응** — `apiRecommend`가 `{lat,lng,areaM2}`만 보내 soil/drainage 공백 → 전국 모든 땅이 동일 순위·신뢰도 ‘D’(가짜 정밀).
  - **비교/PDF 가드레일 누락** — `renderCompare`·인쇄가 면책·출처·신뢰도·손익분기·‘음수=미검증’ 경고를 탈락.

## 4. 우선순위 권고

### P0 — 서사 단절·신뢰 직결 (먼저)
1. **생육·출하를 canonical 경로에 배선** — `ParcelResult`에 재배캘린더(파종·수확월)+생육리스크 필드 추가, `/api/simulate`가 `calendar/growthRisk`를 호출, 소득 카드 아래 **“생육·출하 타임라인” 섹션**(파종→생육리스크→수확월→출하 시점) 노출. *코어는 이미 있음 — 배선·표면화 작업.*
2. **엔진이 입력을 실제로 반영**: `salesChannel`→판로별 단가, `targetYear`→연차별 수량·정착기 현금흐름, **실필지 면적**(폴리곤 면적 계산)→추천·시뮬에 사용.
3. **음수 P50 히어로 카드 문구 개선** — 초보는 빨간 ‘예상 소득 −◯◯◯원’을 “적자”로 오독. ‘미검증 base(placeholder)’임을 카드 본문에서 즉시 구분.

### P1 — 진입·정합·가드레일
4. **진입 마찰 해소**: 주소/지번 검색(`/api/geocode` 배선)+‘내 위치’, 시작 줌 상향 또는 클릭 시 자동 줌인.
5. **토지유형 분기를 시·군 단계에서도**(최소 강/바다 경고) — z=7에서 수면 탭 시 차단 안내.
6. **비교/PDF에 면책·출처·신뢰도·손익분기 포함**(가드레일 일관).
7. **무료 추천 신뢰도·인증게이트를 칩 단계에 노출** + 점수 **범례·‘적합도’ 라벨**(62가 좋은지 나쁜지 알게).

### P2 — 정교화
8. **재확인 실수단**(기존 재배현황 입력) — 현재는 안내 배너뿐. 지목≠실이용 경고 추가.
9. 보정 상태(validated·실측 N)를 **비교표**에 노출, 저장/공유/PDF에 보정 데이터 포함.
