# LEGAL_CHECKLIST.md — 법무·PIPA 컴플라이언스 (상용 런칭 게이트)

> ⚠ **런칭 차단 리스크**: 휴대폰·위치 등 **개인정보 수집이 이미 라이브**인데, 이용약관·개인정보처리방침은 **초안(법무검토 전)**이다(`dashboard/lansmark_privacy.html` "법무 검토 전 실서비스 적용 금지"). 무료 베타라도 PII를 수집·저장하면 「개인정보 보호법(PIPA)」 적용 대상 — **방침 확정 전 공개 수집은 위험**.
> 이 문서는 *실제 코드의 데이터 관행*과 *법무에 넘길 갭*을 정리한다. **법적 확정·계약은 HUMAN GATE(법무·사업자).**

## 1) 실제 수집·저장 개인정보 (코드 기준)
| 항목 | 수집 시점 | 저장 위치 | 코드 근거 |
|---|---|---|---|
| **휴대폰번호** | 알림 옵트인(동의) | `subscriptions.json`(파일·평문) | `notify.ts` `/api/alerts/subscribe`, `alertSubscription.ts` |
| **위치(좌표)·필지** — 분석 | 지도 클릭·시뮬 | **미저장**(분석 후 휘발) | `analysis.ts`(요청 처리만) |
| **위치·수확량·매출** — 재배일지 | '재배 시작' 선택 | `journal.json`(파일·평문·디스크 영속) | `journal.ts`, `FileJournalStore` |
| 익명ID(`anon-*`) | 무료베타 일지·실측 | 위 파일에 동반 | `entitlement.anonSubmitterId` |
| 결제정보 | (유료 시) | **PG사 처리·당사 미보유** | `payment.ts`(Toss seam) |

→ 핵심 PII = **휴대폰번호 + 재배일지(위치·매출)**. 둘 다 파일 평문 저장.

## 2) ✅ 이미 구현된 보호(법무 설명용 근거)
- **수집·이용 동의** 필수 — 동의 없으면 거부(`buildSubscription` CONSENT_REQUIRED) · 동의 시각 기록
- **번호 마스킹** — 로그·응답에 `010****5678`만(`maskPhone`)
- **해지=즉시 파기** — unsubscribe가 레코드 실제 삭제(PIPA 파기의무)
- **가입여부 열거 방지** — 해지 응답 고정 `ok`(존재 노출 차단)
- **최소 수집** — 휴대폰만(이메일·이름·주민번호 미수집)
- **전송 암호화** — HTTPS(nginx TLS), 보안헤더(CSP·HSTS)
- **파일 권한** — PII 파일 `0600`(소유자 전용, `jsonFile.flush`)
- **익명ID 격리** — 일지는 브라우저별 anonId로 사용자 분리(교차열람 차단)

## 3) ❌ 런칭 전 필수 (미충족 — 법무/사업자 액션)
1. **개인정보처리방침·이용약관 확정** — 현재 초안. 법무 검토 → 시행일 명시 → 공개. (`privacy.html`/`terms.html`의 `[ ]` 칸 채우기)
2. **사업자 정보 기입** — 상호·대표자·사업자번호·주소·**개인정보 보호책임자(이름·연락처)**. (방침 `[ ]` 플레이스홀더)
3. ~~**저장 시 암호화(at-rest)**~~ — ✅ **코드 완료(v0.29.0)**: `db/jsonFile.ts` AES-256-GCM seam — `LANSMARK_DATA_KEY`(hex 64자) 설정 시 PII 스토어(휴대폰·일지 좌표/매출) 디스크 암호화(`ENC1:`). 미설정이면 평문+`0600`. **⛔ 남음(HUMAN GATE)**: 운영 키 생성·주입(`openssl rand -hex 32` → 비밀관리자, .env 평문보관·git커밋 금지) + 키 회전 정책.
4. ~~**정보주체 삭제권 UI**~~ — ✅ **코드 완료(v0.29.0)**: 재배일지 "🗑 삭제(파기)" 버튼 + `POST /api/journal/delete`(소유권 검사·즉시 파기). 알림 해지(기존)와 함께 셀프서비스 삭제 충족(열람=일지 조회·정정=작업/수확 수정 기존). ⚠ 익명 보정 레코드(`feedbackStore`)는 **지역단위 가명정보**로 잔존 → 방침에 "실측은 익명집계로 보정 활용, 개별 식별정보는 일지 삭제 시 파기" 명시.
5. **위탁 계약** — SMS 발송사(미선정)·PG사. 방침 '위탁' 표에 수탁자 명시 + 위수탁 계약서.

## 4) ⚠ 권장(런칭 후 단기)
- **백업 정책** — 백업에 PII 포함 → 백업 암호화·접근통제·보유기간. (현재 백업 자체가 없음)
- **anonId 한계 고지** — 일지 격리가 *비암호학적*(추측 불가 의존). 방침/약관에 "무료 베타 일지는 기기 기반 식별" 명시 + 유료(계정) 전환 시 인증 격리.
- **보유기간 구체화** — "서비스 운영 기간"을 구체 기간/파기 시점으로.
- **만 14세 미만** — 수집 제한·법정대리인 동의 정책 명시(농업 특성상 대상 적으나 방침 문구 필요).
- **개인정보 영향평가(PIA)** — 일정 규모 이상 시. 초기엔 해당 낮음, 성장 시 검토.
