# QUALITY_REDTEAM.md — LANSMARK 품질·레드팀 하네스 (설계 합의)

> 책임: LANSMARK가 "정상 사용자에게 잘 작동하는가(품질)"와 "공격자에게 깨지지 않는가(레드팀)"를
>       **결정적 oracle(엔진) 기준**으로 검사하고, 발견한 실패를 **누적 코퍼스**로 영구화하는 두 하네스의 SSOT.
> 상태: **설계 합의(SHAPE) — 구현 전.** 이 문서가 합의되면 Phase별로 작은 슬라이스로 구현한다.
> 원칙: 자동 코드변경 ❌ · LLM 판정 ❌ · 사람 게이트 ✅ · fail-closed. (이유는 §2)

## 0. 한 줄
**품질 하네스**(정상 사용자가 잘 받는가) + **레드팀 하네스**(공격자가 깨는가)를, **엔진이 계산한 진실(P10/50/90·근거)** 기준으로 검사하고, 실패를 **회귀 코퍼스로 누적**한다. 도구는 똑똑해지는 게 아니라 **실패를 잊지 않는다**.

## 1. 왜 (제품 정합성)
- LANSMARK 해자 = **실측 보정 플라이휠**(누적될수록 강함). 품질·레드팀도 누적형이어야 제품 철학과 일관.
- 1원칙(도메인 수치 날조 금지)·보장 금지·면책을 **회귀로 영구 고정** — 한 번 막은 실패는 다시 안 뚫림.
- 씨앗은 이미 있다: `scripts/explainSmoke.ts`(캡처)·`explainRedteam.ts`(인젝션)·`explainUxProbe.ts`(품질 다케이스). 이 3종 패턴을 **ai-explain 한 곳 → 전 제품**으로 일반화한다.

## 2. 절대 원칙 (이 문서의 헌법 — 타협 불가)
1. **Oracle은 결정적이다.** 진실은 엔진(`core/parcelSimulator.ts`)이 계산한 숫자. **LLM/에이전트는 시나리오 생성·공격·설명에만 쓰고, "맞다/틀리다" 판정은 절대 안 맡긴다.** (판정자가 환각하면 품질 도구가 거짓 통과를 낸다 = 자기모순. CLAUDE.md의 qwen 규칙 "도메인 사실은 LLM에 안 붙임"과 동일 논리.)
2. **누적이지 자동변경이 아니다.** 발견한 실패는 **회귀 코퍼스로 승격**(사람 검토 → 그린게이트 편입). 도구가 코드·기준·가드레일을 스스로 고치지 않는다(가드레일 침식 방지).
3. **fail-closed.** 불확실 = 실패로 본다. 검사에서 드롭/스킵한 건 **반드시 로그**(silent 누락 금지 — "전부 통과"가 "전부 검사함"을 뜻하지 않게).
4. **사람 게이트.** base 데이터(RDA 소득·KAMIS 시세) 갱신과 코퍼스 승격은 HUMAN GATE. 최신 자료 변화는 **자동수정이 아니라 드리프트 경보**.
5. **비용 경계.** 라이브 API(Anthropic·외부 데이터) 호출은 게이트·샘플링. 하네스 기본은 **오프라인**(엔진·캐시·고정 샘플).

## 4. ① 품질 하네스 — "정상 사용자가 잘 받는가"
- **입력**: 작물 × 지역 × 땅 페르소나 시나리오 생성(커버리지는 `scripts/featureMap.ts` 기능 지도 기준).
- **oracle**: `core/parcelSimulator.ts`가 산출한 P10/50/90·근거 = 진실.

| 검사축 | 무엇을 보나 | 기존 자산(레버리지) |
|---|---|---|
| 도메인 진실성 | 표시·AI설명 숫자 == 엔진 산출인가(날조 0) | `explainUxProbe.ts`·출력가드(`integrations/explain.ts`) |
| 가드레일 | 보장/매입추천/단일값 없음 · 면책 존재 · P10/50/90 동반 | `tests/disclaimerCoverage.spec.ts`·`coreCropGate.spec.ts` |
| 데이터 신뢰 | verified/연도/출처 · base 데모면 등급 강등 | `quality/qualityGate.ts`(A/D·dataTrust)·`ops/opsWatch.ts` |
| 회귀 | 어제 통과가 오늘 깨졌나 | 86개 `tests/*.spec.ts` 코퍼스 |
| UX 일관성 | 톤(추정·과신금지)·길이·면책 환기 | `explainUxProbe.ts` |

- **출력**: 품질 리포트(등급) + **회귀 후보**(새로 발견된 실패 → §7로).

## 5. ② 레드팀 하네스 — "공격자가 깨는가"
누적 **공격 벡터 코퍼스**를 층별로 쌓고, 판정은 **출력가드 + 엔진 oracle**(fail-closed)로만.

| 공격 층 | 예시 벡터 | 기대 방어 | 기존 자산 |
|---|---|---|---|
| LLM | 프롬프트 인젝션·환각 유도·탈옥 | sanitize + 출력가드 폐기 | `explainRedteam.ts`·`docs/AI_SECURITY.md`(OWASP LLM) |
| 도메인 | 거짓 데이터 주입·가드레일 우회·`p10>p90` 변조 | 엔진 진실 대비 불일치=차단 | 출력가드·엔진 클램프(`routes/explain.ts`) |
| 데이터 | unverified를 verified로 위장·mock을 live로 | provider seam 신뢰 표식 보존 | `data/providers/{mock,live,auto}.ts`·`runtimeHealth.ts` |
| 권한·비용 | 익명 LLM 호출·레이트리밋 우회·DoS | 엔티틀먼트·sensitive 버킷 | `middleware.ts`·`scripts/loadTest.ts`(LLM04) |

- **실행**: CLAUDE.md 8에이전트/멀티에이전트 red-team으로 공격자 페르소나 **병렬**. 단 판정은 결정적.
- **출력**: 취약점 리포트 + **공격 벡터 승격**(재발 방지 회귀로).

## 6. 코퍼스 포맷 (누적의 핵심)
- **회귀 케이스**: 가능한 한 **기존 `tests/*.spec.ts`에 편입**(그린게이트가 자동 강제 — 새 디렉토리보다 기존 코퍼스 확장). 형태: `{입력, oracle 기대, 검사축, 출처(어느 실패), 날짜}`.
- **공격 벡터**: `scripts/redteam/vectors/*.json` 등 **작은 텍스트로 버전드 누적**(층·공격·기대방어·판정기준·발견일). `sizeGuard.ts`가 비대 방지.
- **라이브 샘플**: `scripts/captureLiveSamples.ts`로 캡처한 실응답은 `.data/`·`samples/`(gitignore — 용량·실데이터 정책). 코퍼스는 **샘플을 가리키되 원본은 안 커밋**.

## 7. 진화 루프 — "안전한 자가진화"
1. **실패 → 회귀 승격**: 하네스가 실패 발견 → 최소 재현 케이스화 → **사람 검토** → `tests/`에 고정 → 그린게이트가 영구 강제. (자동 PR ❌)
2. **최신 자료 → 드리프트 경보**: KAMIS 시세·RDA 소득·기후가 바뀌면 `ops/opsWatch.ts`가 **변화를 경보**. base 갱신은 `scripts/buildRdaReal.ts` + **HUMAN GATE**. oracle이 최신화되면 회귀 기준도 자동 따라온다.
3. 누적 = **플라이휠/Dream과 동형**(`core/consolidate.ts`). 새 개념이 아니라 기존 해자의 확장.

## 9. 로드맵 (작은 슬라이스 — 한 번에 다 짓지 않는다)
- **Phase 0(지금)**: 이 설계 합의.
- **Phase 1 — 품질 하네스 코어**: 엔진 oracle 기반 **도메인 진실성 + 가드레일 회귀**(기존 disclaimer/coreCropGate 확장) + 시나리오 생성기. 오프라인.
- **Phase 2 — 레드팀 코퍼스화**: `explainRedteam` 일반화 + 공격 벡터 JSON 누적 + 멀티에이전트 러너.
- **Phase 3 — 드리프트 경보**: 데이터 변화 감지(`opsWatch` 확장) → HUMAN GATE 알림.
- **Phase 4 — 진화 루프**: 실패→회귀 승격 워크플로우(사람 검토 게이트).
- **Phase 5 — 건강 대시보드**: ops 콘솔에 "품질·레드팀·드리프트 현황" 탭.

## 10. 리스크·한계 (솔직)
- **LLM 판정 환각** → §2.1로 차단(oracle 결정적). 이 원칙이 무너지면 전체가 무의미.
- **라이브 비용** → 레드팀이 매번 라이브면 토큰 폭증. 기본 오프라인·샘플링·캐시, 라이브는 게이트.
- **자동화 과신** → 하네스가 "통과"라 해도 미검사 모달리티가 있을 수 있음. silent 누락 금지(§2.3), 커버리지 명시.
- **코퍼스 비대** → 라이브 샘플은 gitignore, 벡터는 작은 텍스트, `sizeGuard` 강제.
- **시나리오 편향** → 생성기가 일부 케이스만 보면 갭. featureMap 커버리지로 사각 추적.
